## 1. Обзор интерфейсов

Бот Криптотрейдер — это **серверное приложение без графического интерфейса** (MVP). Пользователь взаимодействует с ботом через:

| Интерфейс | Тип | Приоритет |
|---|---|---|
| **Telegram-уведомления** | Основной канал информации | 🔴 MVP |
| **CLI (терминал)** | Запуск, остановка, статус | 🔴 MVP |
| **Лог-файлы** | Детальная отладка | 🔴 MVP |
| **Конфиг-файлы** | Настройка параметров | 🔴 MVP |
| **Веб-дашборд** | Мониторинг и управление | 🟢 v2.0 |

## 2. Дизайн Telegram-уведомлений

### 2.1 Принципы UX для уведомлений
- **Сканируемость** — ключевая информация видна с первого взгляда
- **Эмодзи-маркеры** — цветовая кодировка типа события
- **Минимизм** — только важные данные, без «воды»
- **Консистентность** — единый формат для всех типов сообщений
- **Моноширинный шрифт** для чисел — улучшает читаемость

### 2.2 Цветовая кодировка (эмодзи)

| Эмодзи | Значение | Когда |
|---|---|---|
| ✅ | Успех / Старт | Бот запущен |
| 🟢 | Вход в позицию | Открытие ордера |
| 💰 | Прибыль | Сделка закрыта в плюс |
| ⚠️ | Предупреждение | Макс. входов, высокий funding |
| 🔴 | Ошибка | Сбой, разрыв соединения |
| 🛑 | Остановка | Бот остановлен |
| 📊 | Статистика | Ежедневная сводка |
| 🔄 | Процесс | Переподключение |

### 2.4 Форматирование текста

| Элемент | Формат | Пример |
|---|---|---|
| Цены | Доллар + 2 знака | $2,450.30 |
| Объём ETH | 2 знака | 0.20 ETH |
| Проценты | 1 знак + знак | +1.0% |
| PnL | Знак + доллар | +$48.50 |
| Дата | ДД.ММ.ГГГГ | 04.03.2026 |
| Время | ЧЧ:ММ:СС | 15:32:18 |
| Разделитель | Dash-линия | ──────────── |

### 2.5 Правила отправки

| Правило | Описание |
|---|---|
| Без спама | Не более 1 сообщения в минуту (кроме ошибок) |
| Группировка | Если 2+ события за 30 сек — объединить в одно сообщение |
| Тишина ночью | Опциональный «тихий режим» (без звука) с 23:00 до 07:00 |
| Retry | При ошибке Telegram — 3 попытки с задержкой 5 сек |
| Не блокирует | Ошибка Telegram не останавливает торговлю |

## 3. Дизайн CLI-интерфейса

### 3.1 Команды запуска

```bash
# Запуск бота
python main.py start

# Запуск в режиме тестнета
python main.py start --testnet

# Остановка (graceful)

# Проверка статуса
python main.py status

# Просмотр статистики

[2026-03-04 15:32:19] [INFO]  [position]  Ордер размещён: BUY 0.20 ETH @ market
[2026-03-04 15:32:19] [INFO]  [exchange]  Ордер исполнен: ID=abc123, цена=$2,450.30
[2026-03-04 15:32:20] [INFO]  [notifier]  Telegram: уведомление отправлено
[2026-03-04 15:32:20] [INFO]  [database]  Сделка #42 записана в БД
```

### 4.2 Уровни логирования

| Уровень | Что логируется |
|---|---|
| `DEBUG` | Каждый тик цены, детали расчётов, сырые ответы API |
| `INFO` | Сделки, сигналы, соединения, ключевые события |
| `WARNING` | Макс. входов, высокий funding, медленный ответ API |
| `ERROR` | Сбои соединения, ошибки ордеров, недоступен Telegram |
| `CRITICAL` | Ошибка БД, невалидный API-ключ, аварийная остановка |

### 4.3 Ротация логов

| Параметр | Значение |
|---|---|
| Макс. размер файла | 10 МБ |
| Кол-во файлов | 5 (bot.log, bot.log.1, ..., bot.log.4) |
| Формат имени | `bot.log`, `bot.log.1` |
| Хранение | Папка `logs/` |

## 5. Дизайн конфигурации (UX)

### 5.1 Принципы
- **Разделение секретов и настроек**: `.env` (ключи) отдельно от `config.json` (стратегия)
- **Безопасные дефолты**: конфиг работает «из коробки» на тестнете
- **Валидация при старте**: бот проверяет все параметры до начала торговли
- **Понятные ошибки**: при невалидном конфиге — чёткое сообщение что не так

### 5.2 Шаблон `.env.example`

```env
# ═══════════════════════════════════════
# 🤖 Crypto Trader Bot — Конфигурация
# ═══════════════════════════════════════

# ── Bybit API ─────────────────────────
# Создайте API-ключ на https://www.bybit.com/
# Права: ТОЛЬКО торговля (без вывода!)
BYBIT_API_KEY=your_api_key_here
BYBIT_API_SECRET=your_api_secret_here

# Тестовая сеть (true для тестирования)
BYBIT_TESTNET=true

# ── Telegram ──────────────────────────
# Создайте бота через @BotFather
TELEGRAM_BOT_TOKEN=your_bot_token_here

# Узнайте chat_id через @userinfobot
TELEGRAM_CHAT_ID=your_chat_id_here
```

## 6. Веб-дашборд (v2.0 — будущее)

> Не входит в MVP, но описываем для планирования.

### 6.1 Страницы дашборда

| Страница | Содержание |
|---|---|
| **Dashboard** | Текущий статус, баланс, открытая позиция, PnL в реальном времени |
| **Trades** | Таблица всех сделок с фильтрами и сортировкой |
| **Statistics** | Графики доходности (день/неделя/месяц), винрейт |
| **Settings** | Параметры стратегии, управление ботом (старт/стоп) |
| **Logs** | Просмотр логов в реальном времени |

### 6.2 Стиль дашборда

| Параметр | Значение |
|---|---|
| Тема | Тёмная (dark mode) — стандарт для трейдинга |
| Палитра | Тёмно-синий фон, зелёный = прибыль, красный = убыток |
| Шрифт | Inter / JetBrains Mono (для чисел) |
| Фреймворк | React + TailwindCSS (или Next.js) |
| Обновления | WebSocket для реального времени |

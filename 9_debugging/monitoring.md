# 📊 Мониторинг бота

## Логи

### Файловые логи
```
logs/bot.log    # Основной лог (все события)
```

Формат: `[дата] [уровень] [модуль] сообщение`

### Docker логи
```bash
docker compose logs -f              # Все логи
docker compose logs --tail 100      # Последние 100 строк
docker compose logs --since 1h      # За последний час
```

## Telegram-мониторинг

| Команда | Описание |
|---|---|
| `/status` | Баланс, позиция, PnL |
| `/stop` | Остановка бота |
| `/help` | Список команд |

Автоматические уведомления:
- 🟢 Открытие позиции
- 🔵 Усреднение (DCA)
- 💰 Закрытие с прибылью (TP)
- 🛑 Закрытие по стоп-лоссу (SL)
- ❌ Ошибки
- 📊 Ежедневная сводка (21:00)

## CLI-мониторинг

```bash
python main.py status        # Текущая позиция
python main.py stats         # Статистика (всего/винрейт/PnL)
python main.py check-config  # Проверка конфигурации
```

## Health Check

Docker проверяет каждые 60 секунд что SQLite БД доступна. При 3 неудачных проверках контейнер перезапускается.

## 📋 Roadmap

### Фаза A: Исправление багов (1–2 дня)

- [ ] **A-001** Исправить `daily_stats` — бот должен сохранять в БД дневную статистику
- [ ] **A-002** Реализовать `bot_state` — персистенция состояния (last_price, session info)
- [ ] **A-003** Увеличить `recv_window` для Bybit API (проблема синхронизации часов)
- [ ] **A-004** Добавить логирование причин убыточных сделок

### Фаза B: Подготовка к MAINNET (1 день)

- [ ] **B-001** Чеклист безопасности перед MAINNET:
  - API-ключ без прав вывода
  - Whitelist IP на Bybit
  - Минимальный рабочий депозит (~$100)
  - Стоп-лосс подтверждён
- [ ] **B-002** Параметры для MAINNET (пересчёт `working_deposit`)
- [ ] **B-003** Скрипт мониторинга на VPS (uptime, алерты при падении)
- [ ] **B-004** Обновить `.env` на VPS: `BYBIT_TESTNET=false`

### Фаза C: MAINNET запуск (1 неделя)

- [ ] **C-001** Запуск на MAINNET с минимальным депозитом ($50–100)
- [ ] **C-002** Мониторинг первых 48 часов — ручная проверка каждого ордера
- [ ] **C-003** Анализ: совпадают ли результаты Mainnet с Testnet
- [ ] **C-004** Постепенное увеличение `working_deposit` если стабильно

### Фаза D: Улучшения стратегии (v2.1+)

- [ ] **D-001** Trailing Take Profit — вместо фиксированного +1%
- [ ] **D-002** Адаптивный размер входа (от волатильности)
- [ ] **D-003** Фильтр трендовости — не входить против сильного тренда
- [ ] **D-004** Мультипарность (BTC/USDT, SOL/USDT)

## ⚠️ Риски MAINNET

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Слиппедж больше Testnet | Средняя | Средний | Лимитные ордера в будущем |
| Ликвидация при Flash Crash | Низкая | Критический | SL -5%, max 5 входов |

| Потеря VPS | Низкая | Средний | Graceful shutdown + state recovery |

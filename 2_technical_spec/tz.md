## 1. Общее описание

### 1.1 Назначение
Автоматизированный торговый бот, который подключается к бирже **Bybit** через API и торгует парой **ETH/USDT** по лонговой стратегии с усреднением позиций.

### 1.2 Цели проекта
- Полностью автоматическая торговля 24/7 без участия человека
- 100% плюсовых сделок за счёт отсутствия стоп-лосса и ожидания выхода в прибыль
- Целевая доходность ~1% на сделку (~4% с учётом плеча 4x)
- Нулевой риск ликвидации (торговля в пределах депозита, только лонг)

### 1.3 Целевая аудитория
- Владельцы криптовалютных аккаунтов на Bybit
- Пользователи с депозитом от $200
- Люди, ищущие пассивный доход без активного трейдинга

### 1.4 Модель развёртывания
Бот размещается на **отдельном сервере (VPS/VDS)** и работает как самостоятельный сервис 24/7.

- Бот **НЕ устанавливается на локальный компьютер** пользователя (для стабильности 24/7)
- Работает на удалённом сервере (рекомендация: Ubuntu VPS от $5/мес)
- Подключается к бирже Bybit через **API-ключи** (key + secret)
- API-ключ создаётся с правами **только на торговлю** — без прав на вывод средств
- Пользователь заходит на биржу и управляет средствами самостоятельно
- Бот **не имеет доступа к выводу, переводу или управлению аккаунтом**

## 4. Пользовательские истории (User Stories)

### US-01: Первый запуск
> **Как** пользователь, **я хочу** установить бота за 30 минут, **чтобы** он начал торговать автоматически.

**Критерии приёмки:**
1. Инструкция по установке в README
2. Настройка через единый конфигурационный файл
3. Бот запускается одной командой
4. Первая сделка происходит автоматически при наступлении условий

### US-02: Мониторинг прибыли
> **Как** пользователь, **я хочу** видеть статистику сделок в Телеграме, **чтобы** отслеживать прибыль не заходя на биржу.

**Критерии приёмки:**
1. Уведомление при каждом входе и выходе
2. Сумма PnL в уведомлении
3. Ежедневная сводка с итогами дня

### US-03: Безопасность средств
> **Как** пользователь, **я хочу** быть уверен, что бот не сольёт мой депозит, **чтобы** спокойно оставлять его работать 24/7.

**Критерии приёмки:**
1. Бот торгует только LONG
2. Объём всех позиций не превышает баланс
3. Нет цены ликвидации
4. API-ключ без прав на вывод

### US-04: Вывод прибыли
> **Как** пользователь, **я хочу** периодически выводить прибыль, **чтобы** бот работал на фиксированном рабочем депозите.

**Критерии приёмки:**
1. Бот корректно работает при изменении баланса
2. Рабочий депозит настраивается в конфиге
3. Избыточные средства можно выводить без остановки бота

### US-05: Просадка рынка
> **Как** пользователь, **я хочу** чтобы бот продолжал работать при сильном падении ETH, **чтобы** в итоге забрать прибыль.

**Критерии приёмки:**
1. Бот усредняет позицию (до 5 входов)
2. Нет автоматического закрытия в минус
3. Уведомление при достижении максимума входов (5 из 5)
4. Бот продолжает ждать, пока позиция не выйдет в плюс

## 7. Словарь терминов

| Термин | Описание |
|---|---|
| **Лонг (Long)** | Покупка актива с расчётом на рост цены |
| **Шорт (Short)** | Продажа актива с расчётом на падение цены |
| **Усреднение (DCA)** | Добавление к позиции при падении цены для снижения средней цены входа |
| **Тейк-профит** | Автоматическое закрытие позиции при достижении заданной прибыли |
| **Стоп-лосс** | Автоматическое закрытие позиции при убытке (в нашем боте НЕ используется) |
| **Плечо (Leverage)** | Кредитное плечо — умножитель объёма позиции |
| **Ликвидация** | Принудительное закрытие позиции биржей при критическом убытке |
| **Рабочий депозит** | Фиксированная сумма, которая должна быть на аккаунте для работы бота |
| **Funding rate** | Комиссия за удержание позиции (начисляется каждые 8 часов) |
| **PnL** | Profit and Loss — прибыль и убыток |
| **WebSocket** | Протокол для получения данных в реальном времени |
| **REST API** | Интерфейс для отправки торговых команд |
| **Testnet** | Тестовая сеть биржи для безопасного тестирования |

## Модуль 1: Подключение к бирже ✅

**Реализация:** `exchange/client.py`, `exchange/websocket.py`

### F-01: API-авторизация ✅
- **Вход**: API key + API secret из `.env`
- **Действие**: Подключение к Bybit API v5
- **Выход**: Авторизованная сессия
- **Ошибки**: Невалидный ключ → лог + Telegram-уведомление + остановка бота

### F-02: WebSocket-канал ✅
- **Вход**: Авторизованная сессия
- **Действие**: Подписка на тикер ETH/USDT (цена в реальном времени)
- **Выход**: Поток данных о цене
- **Ошибки**: Разрыв соединения → автоматический реконнект (до 10 попыток)

### F-03: Проверка прав API-ключа ✅
- **Вход**: API-ключ
- **Действие**: Проверить, что ключ НЕ имеет прав на вывод
- **Выход**: Подтверждение безопасности / предупреждение
- **Ошибки**: Ключ с правами вывода → WARNING в логах и Telegram

## Модуль 2: Торговый алгоритм ✅

**Реализация:** `trading/strategy.py`, `trading/position_manager.py`

### F-10: Мониторинг цены ✅
- **Вход**: WebSocket-поток ETH/USDT
- **Действие**: Отслеживание цены в реальном времени, расчёт индикатора
- **Выход**: Сигнал «условия для входа выполнены» / «ожидание»

### F-11: Открытие позиции ✅
- **Вход**: Сигнал на вход
- **Действие**: Разместить лимитный/маркет BUY-ордер
- **Выход**: Открытая позиция, запись в лог
- **Проверки**:
  - Кол-во открытых входов < 5
  - Общий объём позиций < баланс
  - Нет цены ликвидации

### F-12: Усреднение ✅
- **Вход**: Открытая позиция + цена ниже на N%
- **Действие**: Добавить BUY-ордер, пересчитать среднюю цену входа
- **Выход**: Обновлённая позиция, новый тейк-профит
- **Ограничения**: Максимум 5 входов

### F-13: Фиксация прибыли ✅
- **Вход**: Текущая цена ≥ средняя цена входа + 1%
- **Действие**: Разместить SELL-ордер (закрыть всю позицию)
- **Выход**: Закрытая сделка, PnL в логах и Telegram

## Модуль 3: Риск-менеджмент ✅

**Реализация:** `trading/risk_manager.py`

### F-20: Контроль ликвидации ✅
- **Перед каждым входом**: Рассчитать будет ли цена ликвидации
- **Если есть цена ликвидации**: Отклонить вход, лог + уведомление

### F-21: Контроль объёма ✅
- **Перед каждым входом**: Проверить что сумма всех позиций ≤ баланс × max_position_pct
- **Если превышает**: Отклонить вход

### F-22: Запрет шорта ✅
- **Жёстко в коде**: Все ордера только side="Buy"
- **Закрытие**: Только через обратный buy → sell (reduce only)

## Модуль 5: Бэктестинг ✅

**Реализация:** `backtester.py`, `download_data.py`, `optimizer.py`

### F-50: Загрузка исторических данных ✅
- **Вход**: Символ, интервал, период
- **Действие**: Скачивание OHLCV свечей из Bybit API
- **Выход**: CSV-файл с данными

### F-51: Симуляция DCA-стратегии ✅
- **Вход**: CSV с данными + параметры стратегии
- **Действие**: Пошаговая симуляция торгового алгоритма (вход, усреднение, тейк-профит)
- **Выход**: Отчёт (кол-во сделок, PnL, макс. просадка, Win Rate)

### F-52: Оптимизация параметров ✅
- **Вход**: Диапазоны параметров (тейк-профит, шаг, плечо и т.д.)
- **Действие**: Перебор комбинаций с оценкой результатов
- **Выход**: CSV с результатами оптимизации

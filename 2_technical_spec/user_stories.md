## US-01: Первый запуск ✅

> **Как** пользователь, **я хочу** установить бота за 30 минут, **чтобы** он начал торговать автоматически.

### Шаги пользователя
1. Клонирует репозиторий
2. Создаёт API-ключ на Bybit (только торговля, без вывода)
3. Копирует `.env.example` → `.env`, вписывает ключи
4. Настраивает `config.json` (депозит, размер позиции)
5. Запускает бота одной командой
6. Получает подтверждение в Телеграм

### Критерии приёмки
- [x] README содержит пошаговую инструкцию
- [x] `.env.example` есть в репозитории
- [x] `config.json` с дефолтными параметрами
- [x] Бот стартует без ошибок
- [x] Первое уведомление в Телеграм: «Бот запущен ✅»

## US-02: Автоматическая сделка ✅

> **Как** пользователь, **я хочу** чтобы бот сам открывал и закрывал сделки, **чтобы** я мог заниматься своими делами.

### Сценарий
1. Бот мониторит цену ETH/USDT
2. Цена падает → срабатывают условия → бот открывает лонг
3. Цена идёт вверх → достигает +1% → бот закрывает с прибылью
4. Пользователь получает уведомление с PnL

### Критерии приёмки
- [x] Сделка открывается автоматически
- [x] Тейк-профит срабатывает автоматически
- [x] Уведомление содержит: цену входа, выхода, PnL, комиссию

---

## US-03: Усреднение при просадке ✅

> **Как** пользователь, **я хочу** чтобы бот усреднял позицию при падении, **чтобы** быстрее выйти в плюс.

### Сценарий
1. Бот открыл 1-ю позицию
2. Цена упала ещё на N% → бот открывает 2-ю позицию
3. Средняя цена входа пересчитывается
4. Повторяется до 5 входов максимум
5. При 5/5 входах — уведомление «Максимум позиций достигнут»

### Критерии приёмки
- [x] До 5 последовательных входов
- [x] Средняя цена пересчитывается корректно
- [x] Тейк-профит от средней цены
- [x] Уведомление при макс. кол-ве входов

## US-06: Проверка стратегии на истории ✅

> **Как** пользователь, **я хочу** протестировать стратегию на исторических данных, **чтобы** убедиться в её прибыльности до торговли реальными деньгами.

### Сценарий
1. Пользователь загружает исторические данные ETH/USDT
2. Запускает бэктест с настраиваемыми параметрами
3. Получает отчёт: количество сделок, PnL, Win Rate, макс. просадка
4. Запускает оптимизацию для поиска лучших параметров

### Критерии приёмки
- [x] Загрузка данных из Bybit API (`download_data.py`)
- [x] Симуляция DCA-стратегии (`backtester.py`)
- [x] Отчёт с PnL, Win Rate, просадкой
- [x] Оптимизация параметров (`optimizer.py`)
- [x] Сохранение результатов в CSV

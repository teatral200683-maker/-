# 📋 Рекомендации по улучшению — Crypto Trader Bot

> **Дата**: 13.03.2026  
> **Версия**: v1.3 → v2.0  
> **Статус**: Бот торгует на Bybit Testnet, VPS

---

## 🔴 Критичные (исправить до MAINNET)

### 1. Персистенция дневной статистики
**Проблема**: Бот не записывает `daily_stats` в SQLite — нет истории балансов по дням.  
**Файл**: `src/bot_engine.py`  
**Решение**: Добавить вызов `db.save_daily_stats()` в конце каждого дня (или при отправке периодической сводки).

### 2. Сохранение состояния бота
**Проблема**: Таблица `bot_state` всегда пуста — при рестарте теряется контекст сессии.  
**Файл**: `src/bot_engine.py`  
**Решение**: Сохранять `last_price`, `session_start`, `total_session_pnl` через `db.save_state()`.

### 3. Синхронизация времени с Bybit
**Проблема**: `recv_window=20000` иногда не хватает — API возвращает ошибку 10002.  
**Файл**: `src/exchange/client.py`  
**Решение**: Использовать NTP-синхронизацию на VPS или увеличить `recv_window` до 30000.

### 4. Логирование убыточных сделок
**Проблема**: Нет анализа WHY — почему конкретная сделка оказалась убыточной.  
**Решение**: Добавить в лог и Telegram-уведомление причину (SL hit, неудачный DCA, etc).

## 🟡 Важные (улучшат работу)

### 5. Trailing Take Profit
**Сейчас**: Фиксированный TP +1% от средней цены.  
**Улучшение**: Trailing TP — при росте цены поднимать TP вслед. На сильных движениях фиксирует больше прибыли.

### 6. Адаптивный размер позиции
**Сейчас**: Фиксированные 5% от рабочего депозита.  
**Улучшение**: Учитывать текущую волатильность (ATR). Выше волатильность → меньше размер.

### 7. Фильтр тренда
**Сейчас**: Бот входит в лонг при любом рынке.  
**Улучшение**: Не входить против сильного нисходящего тренда (EMA 50/200, RSI < 30).

### 8. Скрипт мониторинга на VPS
**Описание**: Watchdog-скрипт, который проверяет работу бота:
- Процесс запущен?
- Последний лог < 5 мин назад?
- WebSocket живой?
- Алерт в Telegram при падении бота.

# 🧪 План тестирования — Bybit Testnet

> **Дата:** 06.03.2026  
> **Цель:** Проверить работоспособность бота на тестовой сети Bybit перед запуском на реальные средства.

---

## Предусловия (для всех тестов)

- [ ] API-ключ Bybit Testnet создан (Read/Write, Orders + Positions)
- [ ] `.env` настроен: `BYBIT_TESTNET=true`
- [ ] Telegram-бот создан, `TELEGRAM_BOT_TOKEN` и `TELEGRAM_CHAT_ID` указаны
- [ ] Зависимости установлены: `pip install -r requirements.txt`
- [ ] На тестовом аккаунте есть баланс USDT (тестовые средства)

---

## T-050: Подключение и базовые проверки

**Цель:** Убедиться, что бот подключается к Bybit Testnet и корректно получает данные.

### Шаги
1. Запустить `python main.py check-config`
2. Запустить `python main.py status`
3. Запустить `python main.py start` и дождаться инициализации

### Ожидаемый результат
- [x] Конфигурация проходит валидацию без ошибок
- [ ] Бот получает баланс USDT с тестового аккаунта
- [ ] Бот получает текущую цену ETH/USDT
- [ ] WebSocket подключается и получает тикер
- [ ] В Telegram приходит уведомление: «Бот запущен ✅»

### Критерий Pass/Fail
**Pass:** Все пункты выполнены, бот работает без ошибок в логе.  
**Fail:** Ошибка подключения, нет данных о балансе/цене, нет Telegram-уведомления.

## T-051: Полный торговый цикл

**Цель:** Проверить цикл: Вход → DCA-усреднение → Take Profit → Закрытие.

### Шаги
1. Запустить бота в режиме Testnet
2. Дождаться срабатывания условий для первого входа
3. Если цена продолжает падать — убедиться, что бот открывает DCA-слои (до 5 входов)
4. Дождаться достижения Take Profit (1% от средней цены)
5. Проверить, что все позиции закрыты

### Ожидаемый результат
- [ ] Первый вход выполнен автоматически
- [ ] Средняя цена пересчитана после каждого DCA-слоя
- [ ] Take Profit обновлён после каждого нового входа
- [ ] Все позиции закрыты при достижении TP
- [ ] PnL рассчитан корректно (с учётом комиссий)
- [ ] Сделки записаны в SQLite базу данных

### Критерий Pass/Fail
**Pass:** Полный цикл Entry → DCA → TP завершён автоматически, PnL > 0.  
**Fail:** Сделка не открылась, TP не сработал, ошибка в расчёте PnL.

## T-052: Telegram-уведомления

**Цель:** Проверить, что все уведомления приходят корректно и содержат нужную информацию.

### Шаги
1. Запустить бота
2. Дождаться входа → проверить уведомление
3. Дождаться выхода → проверить уведомление
4. Дождаться 21:00 → проверить дневную сводку
5. Спровоцировать ошибку (например, отключить интернет) → проверить уведомление об ошибке

### Ожидаемый результат
- [ ] **При входе:** цена входа, размер позиции, номер DCA-слоя, плечо
- [ ] **При выходе:** цена входа, цена выхода, PnL (в $ и %), комиссия
- [ ] **Дневная сводка (21:00):** кол-во сделок за день, общий PnL, текущий баланс
- [ ] **При ошибке:** описание ошибки, статус бота
- [ ] **При макс. позициях (5/5):** уведомление «Максимум позиций достигнут»

### Критерий Pass/Fail
**Pass:** Все 4 типа уведомлений приходят в Telegram с корректными данными.  
**Fail:** Уведомления не приходят, данные неполные или некорректные.

## T-053: Стресс-тест (устойчивость)

**Цель:** Проверить поведение бота при сбоях сети и перезапусках.

### Тест А: Обрыв соединения
1. Запустить бота
2. Отключить интернет на 30 секунд
3. Включить интернет обратно
4. Проверить, что бот автоматически переподключился

### Ожидаемый результат
- [ ] Бот детектирует обрыв соединения
- [ ] Автоматически переподключается (до 10 попыток, интервал 5 сек)
- [ ] Уведомление в Telegram о переподключении
- [ ] Продолжает мониторинг и торговлю после переподключения

### Тест Б: Перезапуск с восстановлением
1. Запустить бота, дождаться открытия позиции
2. Остановить бота (Ctrl+C)
3. Перезапустить бота
4. Проверить, что состояние восстановлено из SQLite

### Ожидаемый результат
- [ ] Graceful shutdown — бот корректно завершает работу
- [ ] При перезапуске — открытые позиции восстановлены из БД
- [ ] Средняя цена и кол-во DCA-слоёв восстановлены
- [ ] Бот продолжает мониторинг от восстановленного состояния

### Критерий Pass/Fail
**Pass:** Авто-переподключение работает, перезапуск восстанавливает состояние.  
**Fail:** Бот не переподключается, теряет состояние при перезапуске, дублирует ордера.

## T-054: Стабильность (24 часа)

**Цель:** Проверить, что бот работает непрерывно 24 часа без ошибок и зависаний.

### Шаги
1. Запустить бота в Testnet-режиме
2. Оставить работать на 24 часа
3. Через 24 часа проверить логи, базу данных и Telegram

### Ожидаемый результат
- [ ] Бот работает 24 часа без крашей
- [ ] WebSocket не теряет соединение (или восстанавливается автоматически)
- [ ] Дневная сводка пришла в Telegram в 21:00
- [ ] Логи не содержат необработанных исключений
- [ ] Потребление памяти стабильно (нет утечек)
- [ ] Все сделки корректно записаны в БД

### Критерий Pass/Fail
**Pass:** 24 часа работы, все сделки проведены, нет ошибок в логах.  
**Fail:** Краш, зависание, утечка памяти, потеря данных.
